# 需求背景

当前项目已经有 `users / roles / permissions / profile / auth` 基础结构，但认证方式仍以用户名密码为主，个人中心也只覆盖基础资料与头像上传。

本次需求要补齐 C 端常见账号能力，并兼容管理端：

- 手机号 + 验证码登录，未注册则直接注册
- 邮箱 + 验证码注册/登录
- 手机号 + 验证码重置密码
- 邮箱 + 验证码重置密码
- 个人信息查询、变更、头像上传

## 目标

1. 保持一套 `users` 主表，不拆两套账号体系。
2. 用端类型、角色、权限区分 C 端与管理端能力。
3. 将验证码、找回密码、上传文件从用户主档中拆出去，降低耦合。
4. 让后续接入短信、邮件、对象存储时可替换、可扩展。

## 非目标

- 不做社交登录
- 不做多租户账号体系
- 不做复杂风控引擎
- 不改现有 RBAC 主链路
